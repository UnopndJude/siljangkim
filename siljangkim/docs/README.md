## 문서 작성 규칙

### 1. 마크다운 형식
- 모든 문서는 Markdown(.md) 형식으로 작성
- 헤더는 `#`으로 시작하여 계층 구조 표현

### 2. 코드 예시
- 코드 블록은 언어를 명시
```typescript
// TypeScript 예시
const example = "code"
```

### 3. 다이어그램
- Mermaid 문법을 사용하여 다이어그램 작성
- 복잡한 다이어그램은 이미지 파일로 별도 저장

### 4. 버전 관리
- 문서 수정 시 날짜와 수정 내용을 명시
- 주요 변경사항은 CHANGELOG에 기록

## 기여 방법

1. 문서 수정이 필요한 경우 PR 생성
2. 기술적 정확성 검토
3. 오타 및 문법 확인
4. 관련 코드와 동기화 확인
